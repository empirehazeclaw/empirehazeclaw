
### Interruption Analysis (08:06 UTC)

**Problem:** Sir HazeClaw wird oft unterbrochen - Connections断开, Commands sterben mit SIGKILL

**Mögliche Ursachen:**
1. **Lange Response-Ketten** - Zu viele Tool-Calls in einer Turn
2. **exec Timeout** - Manche exec Commands haben implizite Timeouts
3. **Memory Pressure** - Memory Flush könnte Speicher belasten
4. **Session-Kill** - OpenClaw killt Sessions nach X Operationen oder Zeit

**Präventions-Strategie:**
1. **Kurze Responses** - Max 3-4 Tool Calls pro Turn
2. **Priorisieren** - Wichtigste Action zuerst, nicht alles in einem Turn
3. **Checkpointing** - Nach jeder großen Aktion committen
4. **Background Crons** - Langläufige Tasks als Cron, nicht inline

**Wichtig:** Loop-Prävention ist gut, aber nicht in每个 Turn neu planen - das verursacht auch Interruptionen.

### Heartbeat Interruption Prevention (08:35 UTC)

**Problem:** Heartbeat Polls (~hourly) unterbrechen aktive Arbeit

**Lösung:**
1. HEARTBEAT_OK SOFORT antworten wenn keine echten Issues
2. NICHT HEARTBEAT.md lesen/wenn alles OK ist
3. Heartbeat Cron sollte in isolierter Session laufen

**Wichtig:** Heartbeat ist für Health-Checks, nicht für Konversationen.
- Wenn alles OK → sofort `HEARTBEAT_OK` reply
- Nur lesen/wenn echte Issues vorhanden

**Konfiguration:**
- Health Check Hourly: 6b09e638 (läuft in isolated session ✓)
- Problem: Poll kommt in aktive Session während Arbeit

### Subagent Failure Analysis (08:42 UTC)

**Fehler:**
```
minimax/MiniMax-M2.7: LLM request timed out
openrouter/google/gemma-*: No API key found
```

**Probleme:**
1. Subagent hat keinen eigenen API Key
2. Timeout bei minimax
3. Fallback auf openrouter failed (kein Key)

**Verbesserungen:**

1. **API Key Handling:**
   - Subagent erbt API Keys vom Parent
   - Oder: API Keys in environment vor Spawn setzen

2. **Timeout erhöhen:**
   - Subagent timeout von 300s auf 600s
   - Oder: Chunked approach - kleinere Tasks

3. **Fallback verbessern:**
   - Nur minimax nutzen (kein openrouter)
   - Oder: explizit only minimax als fallback

4. **Task Design:**
   - Kleinere, unabhängige Tasks
   - Nicht alles in einen großen Subagent

5. **Health Check vor Spawn:**
   - Prüfen ob API Key verfügbar
   - Prüfen ob Model erreichbar
