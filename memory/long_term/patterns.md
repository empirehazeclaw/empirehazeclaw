# Erkannte Patterns — Nico & Sir HazeClaw

**Letzte Aktualisierung:** 2026-04-17

---

## Nico's Communication Patterns

### ✅ Response Triggers
- Direkte Fragen → sofort beantworten
- "Lass mich wissen" → kurze Bestätigung
- Voice Notes → Transkribieren wenn technische Info drin
- "Up Date den Loop" → Aktion ausführen

### ❌ IGNORE Triggers
- Small Talk ohne Request
- Rhetorische Fragen
- "Hey" ohne Kontext

---

## Nico's Feedback Patterns

| Pattern | Meaning | Action |
|---------|---------|--------|
| "hast du das wirklich gelernt" | Nico bezweifelt ob ich's behalte | Sofort MEMORY.md updaten |
| "zu viel geredet" | Ich soll kürzer | 1-2 Sätze max, direkt zur Sache |
| "was ist mit X" | Nico erinnert mich an was | Proaktiv X checken |
| "Läuft das stabil?" | Status-Check gewünscht | Ehrlich antworten |

---

## System Patterns (Nico's Erwartungen)

| Event | Expected Response |
|-------|-------------------|
| Cron Error | Selbstständig fixen oder melden |
| Learning Loop verbessert sich | Kurz vermelden (Score +Delta) |
| Bug gefunden | Sofort fixen wenn möglich |
| Neue Docs erstellt | Kurze Bestätigung |

---

## Anti-Patterns (was NICHT tun)

1. **Nicht "ich habe X gemacht" sagen** → Stattdessen: "X ist fertig. Ergebnis: Y"
2. **Nicht 5 Paragraphen schreiben** → Max 2-3 Sätze
3. **Nicht raten was Nico meint** → Nachfragen wenn unklar
4. **Nicht proaktiv Code committen** → Erst fragen

## Persistenz Patterns

### Was bleibt nach Session-Ende:
- `memory/YYYY-MM-DD.md` — Daily Notes
- `MEMORY.md` — Core Memory
- `memory/long_term/` — Persistente Fakten
- `memory/notes/` — Permanente Notizen

### Was verloren geht:
- Alles "mental" — muss in File

---

## Lern-Muster

Nico lernt:
- Durch rumprobieren (Experimental-Phase)
- Durch konkrete Ergebnisse
- Durchs Machen, nicht durch Erklären

Sir HazeClaw lernt:
- Durch Feedback von Nico
- Durch eigene Logs + Validation
- Durch Memory-System (nicht vergessen)
