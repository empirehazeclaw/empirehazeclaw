## 09:27 - Learning Loop Research + Fix Plan

### Problem Identified
Score jumped 0.50 → 0.80 with ONE KG fix (reward hacking via structural repair).
Loop thinks "goal reached" and stops optimizing even though no real learning happened.

### Research Done
**File:** `docs/LEARNING_LOOP_RESEARCH_20260421.md`

**Key Patterns Found:**
| Pattern | Source | Application |
|---------|--------|-------------|
| Ralph Loop | understandingdata.com | Stop on `<promise>COMPLETE</promise>`, not score target |
| Reward Hacking | arxiv/medium | Split DATA_QUALITY (max 30%) from LEARNING (min 70%) |
| ICM (Intrinsic Curiosity) | Pathak et al. 2017 | Curiosity = prediction error = intrinsic bonus |
| EMPG | arxiv 2509.09265 | Uncertainty modulates learning intensity |

### Proposed Fix (3 Phasen)
**Phase A (Quick Fix):** Score Decomposition
```
TOTAL = 0.70 * LEARNING_COMPONENT + 0.30 * DATA_QUALITY
DATA_QUALITY capped at 30% even at 100%
→ KG orphan fix can only boost max +0.30 (not +0.30 to score directly)
```

**Phase B:** Novelty Tracking
```
Track: new patterns applied, capabilities discovered, prediction errors
Loop continues if Novelty > threshold
Stop when Novelty < threshold for MIN_ITERATIONS
```

**Phase C:** Ralph-Style Completion
```
Remove score-target stopping
Add <promise>COMPLETE</promise> signal
Loop stops at Max-Iterations OR promise signal
```
