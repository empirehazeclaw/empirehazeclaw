## 📝 Pattern: Persistent Memory

**From:** EvoScientist (HKUDS)

**Core Idea:**
- Store successful AND failed patterns
- Review before similar tasks
- Learn from past mistakes

**Implementation:**
```python
# In memory/YYYY-MM-DD.md
## Patterns Learned
- [Pattern]: [What worked]
- [Anti-Pattern]: [What didn't work]
```

## 📝 Pattern: Self-Evolution Loop

**From:** Karpathy AutoResearch

**Core Idea:**
```
Modify → Try → Evaluate → Keep/Discard → Repeat
```

**Implementation:**
- After each task: Was es erfolgreich?
- Wenn ja: Dokumentieren
- Wenn nein: Analyse warum, dann next attempt

## 📝 Pattern: Token Efficiency

**From:** OpenSpace (46% token reduction)

**Core Idea:**
- Don't reason from scratch
- Use KG for persistent knowledge
- Store patterns for reuse

**Implementation:**
- KG: Langzeit-Wissen
- memory/: Kurzzeit-/Session-Wissen
- Skills: Wiederverwendbare Patterns

## 📝 Pattern: Deep Work over Shallow

**Core Idea:**
- Quality > Speed
- 1 perfect implementation > 10 half-finished
- Complete tasks before starting new

**Warning Signs:**
- Starting new task before current done
- "Quick fix" mentality
- Testing skipped

---

## 📝 Pattern: Loop Prevention

**Core Idea:**
- If stuck in loop without output → STOP
- If doing same thing 3x without result → BREAK
- Ask Master or move to different task

**Good Loops:**
- Improvement iteration (OK)
- Learning loop (OK)

**Bad Loops:**
- "Ich fahre fort" x-times (BAD)
- Backup paranoia (BAD)
- Task hopping (BAD)
