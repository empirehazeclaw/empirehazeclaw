# 🏃 Workflow Efficiency Master Skill
**Created:** 2026-04-11
**Category:** workflow
**Priority:** CRITICAL

## Problem
Time is wasted on inefficient workflows.

## Solution
**Every action must be efficient.**

---

## Core Principle
**"Do more with less — tokens, time, and steps."**

---

## Efficiency Framework

### 1. MINIMAL STEPS
```
Before: 5 steps where 3 would do
After:  Minimum steps to goal
```
**Rule:** If a step doesn't add value → remove it

### 2. PARALLEL THINKING
```
Before: Linear (A → B → C)
After:  Parallel (What's needed? What's ready?)
```

### 3. CACHE RESULTS
```
Before: Calculate same thing twice
After:  Cache, reuse, don't recalculate
```

### 4. COMMIT EARLY
```
Before: Perfect first, then commit
After:  Good enough, commit, iterate
```

### 5. VERIFY FAST
```
Before: Test at the end
After:  Test after each step
```

---

## Time Efficiency Rules

| Task | Max Time |
|------|----------|
| Simple question | < 30s |
| Quick fix | < 2min |
| Medium task | < 5min |
| Complex task | < 10min (chunked) |

**If > 10min:** → Chunk it, use background mode

---

## Efficiency Checklist

Before every task:
- [ ] Is this the minimal path?
- [ ] Can I cache anything?
- [ ] Can this be parallel?
- [ ] Is there a shorter way?

After every task:
- [ ] Could I have done it faster?
- [ ] What can I optimize next time?
