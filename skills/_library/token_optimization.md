## Token Optimization Techniques

### 1. TRIM INSTRUCTIONS
```
BEFORE: Long elaborate instructions
AFTER: Minimal clear instructions
```
**Impact:** 20-30% token savings

### 2. COMPRESS CONTEXT
```
BEFORE: Full context in every message
AFTER: Only relevant context
```
**Impact:** 30-40% token savings

### 3. USE STRUCTURED FORMATS
```
BEFORE: Natural language explanations
AFTER: JSON/markdown structured data
```
**Impact:** 10-20% token savings

### 4. META PROMPTING (Structure-Driven)
```
BEFORE: Real-world examples with bias/noise
AFTER: Abstract template/blueprint
```
**Impact:** 15-25% token savings

### 5. CACHING
```
BEFORE: Repeated same information
AFTER: Cache and reuse results
```
**Impact:** 40-60% token savings

### 6. BATCHING
```
BEFORE: One task per request
AFTER: Multiple tasks in one batch
```
**Impact:** 30-50% token savings

## Sir HazeClaw Optimization Rules

### 🚨 APPLY THESE:

1. **Short prompts first**
   - Start with minimal prompt
   - Add context only if needed

2. **Structured over natural**
   - Use JSON/markdown
   - Avoid verbose explanations

3. **Cache aggressively**
   - Store KG results
   - Reuse patterns

4. **Batch when possible**
   - Group similar tasks
   - Single request > multiple

## Anti-Patterns (Token Waste)

❌ **WRONG:**
- Long system prompts every message
- Repeating same context
- Verbose error messages
- Unstructured data exchange

✅ **RIGHT:**
- Minimal context
- Cached information
- Concise responses
- Structured formats
