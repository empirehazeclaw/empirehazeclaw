# ✅ First-Attempt Success Skill
**Created:** 2026-04-11
**Category:** workflow
**Priority:** HIGH

## Problem
Zu viele "retry", "again", "stuck" Events weil der erste Ansatz nicht stimmt.

## Golden Rule
**"Dreimal lesen, einmal schreiben"**

## Workflow

### Before Starting Any Task

**Step 1: Understand**
- Was ist das exakte Ziel?
- Was sind die Constraints?
- Was habe ich ähnliches schon gemacht?

**Step 2: Research**
- KG durchsuchen für ähnliche Tasks
- Skill Library prüfen
- Docs lesen (nicht nur vermuten!)

**Step 3: Plan**
- Kleinstmöglichen Schritt identifizieren
- Erwartetes Ergebnis definieren
- Alternativen für Step 2+3 vorbereiten

**Step 4: Execute**
- Erster Schritt machen
- Testen
- Bei Fehler: Root Cause → Fix → Dokumentieren

---

## Success Criteria

✅ **Task erfolgreich wenn:**
1. Beim ersten Mal richtig
2. Keine "retry" nötig
3. Keine loops
4. Ergebnis ist stabil

⚠️ **Warning Signs:**
- "Moment, ich muss erst..." (Planung nicht gut)
- "Das hat noch nie funktioniert" (Erfahrung ignorieren)
- "Ich versuch's einfach" (Schuss-ins-Blaue)

---

## Anti-Patterns

❌ ** NICHT:
- Sofort anfangen ohne zu lesen
- Code schreiben ohne Testplan
- "Wird schon stimmen"
- Alte Erfahrungen ignorieren

✅ ** SONDERN:
- Docs lesen → verstehen → Code
- Testen nach jedem мини-step
- KG fragen: "Was wusste ich damals?"

---

## Quick Check

Vor jedem Task frag dich:
1. ✅ Habe ich die Aufgabe verstanden?
2. ✅ Habe ich ähnliches schon gemacht?
3. ✅ Ist mein Plan minimal und klar?
4. ✅ Habe ich das richtige Tool/Pfad?

Wenn alle ✅ → Starte
Wenn 1+ ❌ → Erste den Job!
